🚀 Task Management API

RESTful API for task management with authentication, categories, and real-time updates

Base URL: https://humane-properly-bug.ngrok-free.app
Overview Authentication Auth Endpoints Task Endpoints Category Endpoints Data Models Error Codes

📋 Overview

This is a RESTful API for a task management system built with Node.js, Express, and PostgreSQL. The API supports user authentication, task management, and category organization with a clean, consistent interface.

✨ Features

  • 🔐 JWT-based authentication with refresh tokens
  • 📝 Complete CRUD operations for tasks and categories
  • 🔍 Advanced filtering, sorting, and pagination
  • 📊 Task statistics and analytics
  • 🎨 Color-coded categories
  • ⚡ Real-time data validation

🔐 Authentication

The API uses JWT (JSON Web Tokens) for authentication. Include the token in the Authorization header for protected endpoints:

Authorization Header: Authorization: Bearer <your_jwt_token>

📦 Common Response Format

{
  "success": boolean,
  "message": "string",
  "data": {} // Response data (when applicable)
}

❌ Error Response Format

{
  "success": false,
  "message": "Error description",
  "details": ["validation error details"] // For validation errors
}

🔑 Authentication Endpoints

POST Register User

/api/auth/register

Description: Create a new user account

Request Body:

{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "password123"
}

Validation Rules:

Field Type Requirements
name string Required, 2-50 characters
email string Required, valid email format
password string Required, minimum 6 characters

201 Success Response:

{
  "success": true,
  "message": "User registered successfully",
  "data": {
    "user": {
      "id": "cm123abc",
      "name": "John Doe",
      "email": "john@example.com",
      "createdAt": "2025-06-21T10:00:00.000Z"
    },
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs..."
  }
}

POST Login User

/api/auth/login

Description: Authenticate user and receive access tokens

Request Body:

{
  "email": "john@example.com",
  "password": "password123"
}

POST Refresh Token

/api/auth/refresh

Description: Get new access token using refresh token

GET Get Current User

/api/auth/me

Description: Get current authenticated user information

Headers: Authorization: Bearer <token>

📋 Task Endpoints

GET Get All Tasks

/api/tasks

Description: Get paginated list of user's tasks with filtering and sorting

Query Parameters:

Parameter Type Default Description
page number 1 Page number for pagination
limit number 10 Number of tasks per page
status string - Filter by task status (PENDING, IN_PROGRESS, COMPLETED, CANCELLED)
categoryId string - Filter by category ID
search string - Search in title and description
sortBy string createdAt Field to sort by
sortOrder string desc Sort order (asc or desc)

Example Request:

GET /api/tasks?page=1&limit=5&status=PENDING&search=project

GET Get Single Task

/api/tasks/:id

POST Create Task

/api/tasks

Description: Create a new task

Request Body:

{
  "title": "New Task",
  "description": "Task description (optional)",
  "dueDate": "2025-06-30T15:00:00.000Z",
  "status": "PENDING",
  "categoryId": "work-category"
}

PUT Update Task

/api/tasks/:id

DELETE Delete Task

/api/tasks/:id

🎨 Category Endpoints

GET Get All Categories

/api/categories

Description: Get all categories for the authenticated user with task counts

GET Get Single Category

/api/categories/:id

POST Create Category

/api/categories

Description: Create a new category

Request Body:

{
  "name": "Health",
  "color": "#F59E0B"
}

Validation Rules:

Field Type Requirements
name string Required, 1-50 characters, must be unique per user
color string Optional, hex color format (default: "#3B82F6")

PUT Update Category

/api/categories/:id

DELETE Delete Category

/api/categories/:id

Note: Only categories with no associated tasks can be deleted.

📊 Data Models

👤 User

id: string // Unique identifier
email: string // User email (unique)
name: string // User display name
createdAt: Date // Account creation timestamp
updatedAt: Date // Last update timestamp

📝 Task

id: string // Unique identifier
title: string // Task title
description?: string // Optional task description
dueDate?: Date // Optional due date
status: TaskStatus // PENDING | IN_PROGRESS | COMPLETED | CANCELLED
userId: string // Owner user ID
categoryId?: string // Optional category ID
createdAt: Date // Creation timestamp
updatedAt: Date // Last update timestamp
category?: Category // Associated category (when included)

🎨 Category

id: string // Unique identifier
name: string // Category name
color: string // Hex color code
userId: string // Owner user ID
createdAt: Date // Creation timestamp
updatedAt: Date // Last update timestamp
_count?: { tasks: number } // Task count (when included)
tasks?: Task[] // Associated tasks (when included)

⚠️ Error Codes

200
OK - Request successful
201
Created - Resource created successfully
400
Bad Request - Invalid input data
401
Unauthorized - Invalid or missing token
404
Not Found - Resource doesn't exist
500
Internal Server Error

🚀 Quick Start

Get started with the Task Management API in just a few steps:

1️⃣ Register a New User

curl -X POST https://humane-properly-bug.ngrok-free.app/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Your Name",
    "email": "your.email@example.com",
    "password": "securepassword123"
  }'

2️⃣ Create Your First Task

curl -X POST https://humane-properly-bug.ngrok-free.app/api/tasks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -d '{
    "title": "My First Task",
    "description": "Getting started with the API",
    "status": "PENDING"
  }'

3️⃣ Create a Category

curl -X POST https://humane-properly-bug.ngrok-free.app/api/categories \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -d '{
    "name": "Work",
    "color": "#3B82F6"
  }'

4️⃣ Get Your Tasks

curl -X GET https://humane-properly-bug.ngrok-free.app/api/tasks \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"

💡 Tips & Best Practices

🔒 Security

Always include the Authorization header for protected endpoints. Store tokens securely and refresh them when needed.

📊 Pagination

Use pagination for large datasets. Start with reasonable limits (10-50 items) and implement client-side caching.

🎨 Categories

Use color-coded categories to organize tasks visually. Stick to a consistent color palette for better UX.

⚡ Performance

Use filtering and sorting parameters to reduce payload size. Cache frequently accessed data on the client side.
